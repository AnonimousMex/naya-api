# Contributing to Nayá E2E Tests\n\nGracias por tu interés en contribuir a la suite de pruebas E2E de Nayá. Este documento proporciona directrices para contribuir.\n\n## 🤝 Cómo Contribuir\n\n### Reportar Bugs\n\nAl crear un reporte de bug, incluye:\n- Descripción clara y concisa\n- Pasos para reproducir\n- Comportamiento observado\n- Comportamiento esperado\n- Capturas de pantalla si es posible\n- Información del sistema (OS, Node.js, npm)\n\n### Sugerir Mejoras\n\nAl sugerir mejoras:\n- Usa un título claro y descriptivo\n- Proporciona descripción detallada\n- Lista algunos ejemplos específicos para demostrar el caso de uso\n- Nota en la lista los proyectos similares\n\n### Pull Requests\n\n1. Fork el repositorio\n2. Crea una rama para tu feature (`git checkout -b feature/AmazingFeature`)\n3. Commit tus cambios (`git commit -m 'Add some AmazingFeature'`)\n4. Push a la rama (`git push origin feature/AmazingFeature`)\n5. Abre un Pull Request\n\n## 📝 Directrices de Código\n\n### TypeScript\n\n- Usa tipos explícitos\n- Evita `any`\n- Sigue las convenciones de nombres\n- Documenta funciones complejas\n\n### Tests\n\n- Cada test debe ser independiente\n- Usa descriptores claros\n- Agrupa tests relacionados\n- Incluye tanto casos positivos como negativos\n\n### Nuevos Flujos\n\nSi agregas un nuevo flujo:\n\n1. Crea test en `tests/new-feature.spec.ts`\n2. Actualiza `FLOWS_DOCUMENTATION.md`\n3. Actualiza `README.md`\n4. Agrega test data en `test-data.json`\n\n## 🧪 Testing\n\nAntes de hacer PR:\n\n```bash\nnpm test                   # Ejecuta todos los tests\nnpm run test:debug        # Debug de tests\nnpm run test:report       # Ver reporte\n```\n\n## 📖 Documentación\n\nMantén estos archivos actualizados:\n\n- `README.md` - Información general\n- `EXECUTION_GUIDE.md` - Cómo ejecutar\n- `FLOWS_DOCUMENTATION.md` - Detalles de flujos\n\n## 🎯 Estándares de Calidad\n\n- ✅ Todos los tests deben pasar\n- ✅ Cobertura mínima del 80%\n- ✅ Sin warnings de TypeScript\n- ✅ Código documentado\n\n## 📞 Contacto\n\nPara preguntas, abre un Issue o contacta al equipo.\n\n---\n\nGracias por contribuir! 🙌\n
